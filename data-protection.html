Data Protection

This page summarises the current data-minimisation approach used by the public build.

Home / Data Protection

The public build is designed to minimise data handling. It prefers outward links, public information, and email-first contact rather than collecting large amounts of personal data through web forms or public community features.

How protection is applied

  • Only proportionate data needed for ordinary correspondence, correction handling, or basic site operation should be processed.
  • Public browser moderation tools and public sign-up features are disabled in the public build.
  • Theme choice, cookie choice, and similar interface preferences may be stored locally in the browser.
  • Where email correspondence is received, it should be handled only by those responsible for the project and only for the purpose for which it was sent.

Requests and concerns

  • For data rights requests or concerns, contact the site contact address the published site contact route once it is configured.
  • If the site later introduces newsletters or managed forms, this page should be reviewed and updated before those tools are opened publicly.
  • Safeguarding or urgent risk concerns should be directed to the proper civil or Church authority rather than handled as ordinary website correspondence.
