# The Catholic Experiment — Common Platform

This directory is the controlled source for programme-wide website standards.

## Approved primary navigation

### Central Catholic Experiment
Home / Courses / Schedule / Materials / Certificates / Contact

### Individual Experiments
Home / Course / Schedule / Materials / Certificates / Contact

The labels and order are fixed. Experiment-specific items such as Journal, Grammar, Progress and Videos belong within a primary page or secondary course navigation.

## Current repository family
- Central: `Wally189/The-Catholic-Experiment`
- Latin: `Wally189/the-latin-experiment`
- Faith: `Wally189/The-Faith-Experiment`

All currently use the `main` branch. Latin and Faith are static HTML/CSS/JavaScript sites. The central repository is the programme control and future central-site source.

## Change sequence
1. Build and approve the central shell here.
2. Test mobile, desktop and keyboard behaviour.
3. Apply the same navigation rules to Latin without changing lesson content.
4. Apply the same navigation rules to Faith, moving Journal out of the primary menu and adding Certificates.
5. Record release versions and test results in the programme Release and Evaluation Record.

## Minimum recovery procedure
1. Sign in to the GitHub account controlling `Wally189`.
2. Clone or download the relevant repository from the `main` branch.
3. Confirm `index.html` and linked local assets are present.
4. Reconnect the hosting service to the repository and deploy the repository root as a static site.
5. Test every primary navigation item, lesson link, contact route and external source link.
6. Record the restored URL, provider, date and tester in the Common Platform Implementation Appendix.

Hosting provider, live URL, domain/DNS ownership and deployment trigger remain to be recorded before Phase 4.1 can close.
